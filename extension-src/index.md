# Extension source index

TypeScript source tree and static assets for the GNOME Shell extension. This is
the source of truth; `../extension/` is generated by `npm run build`.

## Directories

- [`plugins/`](plugins/index.md) ‚Äî built-in panel widgets, one directory per
  widget.
- `schemas/` ‚Äî GSettings schema copied into generated extension and compiled by
  `install.sh`.

## Core modules

- `extension.ts` ‚Äî GNOME Shell extension entrypoint and floating panel actor.
- `prefs.ts` ‚Äî preferences UI to add, remove, reorder, enable and configure
  widgets; see [`../docs/preferences.md`](../docs/preferences.md).
- `pluginManager.ts` ‚Äî reads widget config and instantiates plugin actors.
- `mainPanel.ts` ‚Äî `MainPanelController`: hides/auto-hides the GNOME top bar
  (`panelBox`) per the `main-panel` GSettings enum; a built-in replacement for
  the "Hide Top Bar" extension (`@tag:main-panel`).
- `configStore.ts` ‚Äî reads/writes the `widgets` GSettings key (including
  one-time legacy `widgets.json` migration); delegates parsing to
  `widgetConfig.ts`.
- `widgetConfig.ts` ‚Äî gi-free parse/validate/serialize of the config document
  stored in the `widgets` key (unit tested in [`../tests/`](../tests/index.md)).
- `contracts.ts` ‚Äî typed contracts shared by host, registry and preferences.
- `tooltipTemplate.ts` ‚Äî shared `renderTemplate` for user-editable Pango-markup
  hover tooltips; no `gi://` import, so it loads in both the Shell and the
  preferences process (`@tag:ui`).
- `colorUtils.ts` ‚Äî gi-free colour/numeric helpers (`hexToRgb`, `toNumber`,
  `nowSeconds`) shared by the graph widgets (unit tested in
  [`../tests/`](../tests/index.md)).
- `tooltip.ts` ‚Äî shared hover-tooltip positioning (`positionTooltip`) and
  show/hide animation (`animateTooltipVisibility`) used by the graph widgets.
- `prefsColor.ts` ‚Äî shared preferences colour helpers (`hexToRgba`,
  `rgbaToHex`, unified `colorButton` factory) used by the widget prefs pages.
- `prefsTemplate.ts` ‚Äî shared preferences tooltip-template editor
  (`addTemplateEditor`) used by the graph-widget prefs pages.
- `controlButton.ts` ‚Äî panel handle, menu (Settings‚Ä¶, About) and movement
  controls.
- `systemInfo.ts` ‚Äî shared helper: collects a best-effort system report, builds
  prefilled GitHub issue-form URLs (bug/feature/widget request) and opens URLs;
  runs in both the Shell and preferences process (`@tag:mechanism`).
- `indicatorsDrawer.ts` ‚Äî reusable GNOME indicator clone drawer.
- `utils.ts` ‚Äî monitor/panel helper functions.

The plugin metadata registry used by preferences is
[`plugins/registry.ts`](plugins/registry.ts).

See [`../docs/object-model.md`](../docs/object-model.md) for runtime objects.
