# Extension source index

TypeScript source tree and static assets for the GNOME Shell extension. This is
the source of truth; `../extension/` is generated by `npm run build`.

## Directories

- [`plugins/`](plugins/index.md) — built-in panel widgets, one directory per
  widget.
- `schemas/` — GSettings schema copied into generated extension and compiled by
  `install.sh`.

## Core modules

- `extension.ts` — GNOME Shell extension entrypoint and floating panel actor.
- `prefs.ts` — preferences UI to add, remove, reorder, enable and configure
  widgets; see [`../docs/preferences.md`](../docs/preferences.md).
- `pluginManager.ts` — reads widget config and instantiates plugin actors.
- `mainPanel.ts` — `MainPanelController`: hides/auto-hides the GNOME top bar
  (`panelBox`) per the `main-panel` GSettings enum; a built-in replacement for
  the "Hide Top Bar" extension (`@tag:main-panel`).
- `configStore.ts` — reads/writes the `widgets` GSettings key (including
  one-time legacy `widgets.json` migration); delegates parsing to
  `widgetConfig.ts`.
- `widgetConfig.ts` — gi-free parse/validate/serialize of the config document
  stored in the `widgets` key (unit tested in [`../tests/`](../tests/index.md)).
- `contracts.ts` — typed contracts shared by host, registry and preferences.
- `tooltipTemplate.ts` — shared `renderTemplate` for user-editable Pango-markup
  hover tooltips; no `gi://` import, so it loads in both the Shell and the
  preferences process (`@tag:ui`).
- `colorUtils.ts` — gi-free colour/numeric helpers (`hexToRgb`, `toNumber`,
  `nowSeconds`) shared by the graph widgets (unit tested in
  [`../tests/`](../tests/index.md)).
- `tooltip.ts` — shared hover-tooltip positioning (`positionTooltip`) and
  show/hide animation (`animateTooltipVisibility`) used by the graph widgets.
- `prefsColor.ts` — shared preferences colour helpers (`hexToRgba`,
  `rgbaToHex`, unified `colorButton` factory) used by the widget prefs pages.
- `prefsTemplate.ts` — shared preferences tooltip-template editor
  (`addTemplateEditor`) used by the graph-widget prefs pages.
- `controlButton.ts` — panel handle, menu (version header, Settings…, Release
  notes, extensions.gnome.org, Report a bug) and movement controls.
- `version.ts` — gi-free release-channel (`alpha`) badge constant and
  version-label formatting shared by the menu, About group and bug reports; unit
  tested in [`../tests/`](../tests/index.md) (`@tag:versioning`; see
  [`../docs/release.md`](../docs/release.md)).
- `systemInfo.ts` — shared helper: reports the extension version
  (`versionName`/`versionDisplay`), builds the current-version release-notes URL
  (`releaseNotesUrl`), changelog URL and the extensions.gnome.org store URL
  (`egoUrl`), collects a best-effort system report, builds prefilled GitHub
  issue-form URLs (bug/feature/widget request) and opens URLs; runs in both the
  Shell and preferences process (`@tag:mechanism`).
- `indicatorsDrawer.ts` — reusable GNOME indicator clone drawer.
- `utils.ts` — monitor/panel helper functions.

The plugin metadata registry used by preferences is
[`plugins/registry.ts`](plugins/registry.ts).

See [`../docs/object-model.md`](../docs/object-model.md) for runtime objects.
